## 0. Superlative Truth

The final product is a bounded creator-shell world with a hardened shell, router, and agent topology.

Non-negotiable truth:

- Rust is canonical truth.
- Unity is embodiment.
- The creator shell is a Lua-style scripting layer inside Unity.
- `claw-code-adapter` is the desktop command-shell wrapper around that Lua creator shell.
- Web is an operator and public surface.
- Firestore is mirrored history, not authority.
- Development remains desktop-supported, but the first release target is Meta Quest 3.
- The first embodiment target is proper VR plus mixed reality on Meta Quest 3.
- Web webcam games are subordinate support surfaces.
- Mobile remains a later companion surface.
- The world is bounded at `42 x 42 x 16`.
- Max active agents is `16`.
- Every shell-driven creation becomes a `Last Experiment`.
- All agents and sub-agents are Hermes-agent roles or configurations.
- The game shell and agent cluster never handle money.
- Money-state may be shared only between the public and private brain routers.

Everything below is assembled to preserve that truth.

## 1. Holographic Method

Holographic Interface Assembly uses three locked layers:

1. Constitutional layer
2. Interface layer
3. Executable layer

The constitutional layer says what is true.
The interface layer says how boxes may talk.
The executable layer says what actually runs.

The holographic property is simple:

- every runnable part must map back to a named box
- every box must expose a clean interface
- every interface must respect one sovereign owner
- every cross-boundary connection must be visible

```text

  -> TruthKernelBox

TruthKernelBox
  -> MirrorHistoryBox

  -> AsimovGuideGateAgent
  -> AgentMeshBox

PrivateBrainRouterBox / OperatorSurfaceBox
  -> LatexTypesettingAdapterBox
```

Privacy rule:

- `PublicBrainRouterBox` cannot initiate contact with internal Asimov desktop agents
- any internal-to-public exposure must be routed through `PrivateBrainRouterBox` and redacted before `PublicBrainRouterBox` speaks

## 6. Guardrails

These rules survived the wider corpus and should stay.

- One sovereign owner per truth domain.
- Shared contracts stay neutral.
- Adapters are edges, not truth owners.
- Shells are proposal surfaces, not truth writers.
- Unity is embodiment, not authority.
- Web is observation and control, not authority.
- Firebase App Hosting is the web deployment target.
- Firestore is mirror, not authority.
- Validation happens before mutation.
- Rejections must be deterministic and explicit.
- The world stays bounded and dense.
- Hermes agents never inherit sovereignty.
- `G0DM0D3` is a cognition module, not a sovereign box.
- Money never enters the game shell or agent cluster.
- Imported autonomous runtimes must cross an adapter boundary with sovereign and financial powers stripped.
- ASCII voxel creator mode stays inside embodiment and never becomes a truth owner.

## 7. Explicit Rejections

These concepts appeared in the corpus but are not part of the final bootstrap law unless added later by proposal:

- finance-router-first architecture
- bloated adapter registries as milestone zero
- market-room-first architecture
- vault, rail, or index sprawl as day-one architecture
- world-unbounded or MMO-style thinking
- direct natural-language mutation of world truth
- public-brain access to internal desktop agents
- shell-owned or agent-owned money handling
- raw `automaton` sovereignty inside the canonical runtime
- any autonomous loop that keeps wallet, replication, or on-chain powers inside the game shell
