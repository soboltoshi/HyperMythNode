# Assembly Assimilation

Assembly Assimilation is the repo-local skill that turns a foreign repo, local folder, or service surface into Interface Assembly language.

Use it when you want to answer:

- What is this repo, structurally?
- Is it a box, an adapter, a shared interface, or a sidecar?
- What are its primitives?
- What does it depend on?
- How should it enter the world?

## Commands

- `ASSIMILATE://GITHUB+REPO`
- `ASSIMILATE://LOCAL+FOLDER`
- `ASSIMILATE://SERVICE+API`
- `ASSIMILATE://MODEL+STACK`

## Output

Every assimilation should emit:

- assimilation report
- box map
- runtime contract
- environment contract
- mesh role
- risk register
- interface URL set
- adoption recommendation

## Important rule

Assimilation makes foreign systems legible. It does not automatically make them native.
