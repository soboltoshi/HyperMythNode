
# Meta Quest Submission Track

This repo includes Quest submission-readiness planning for the VR shell.

## Why this exists

The VR Superbrain may be submitted later as a Quest-facing experience. The official Meta developer documentation groups Quest requirements under multiple VRC and policy areas, including packaging, audio, performance, functional, security, tracking, input, asset, ads, accessibility, streaming, privacy, developer policies, Android manifest checks, build upload, release channels, app metadata, and signing/submission steps. These categories should be tracked from day one in repo planning.

## Repo rule

The Quest-facing shell stays separate from the desktop execution shell.
VR handles:
- perception
- world presence
- VR interaction
- immersive cognition
- safe command issuance

Desktop handles:
- wallet control
- crypto execution
- market routing
- device control
- privileged operations
