## How The Hub Works

The hub mirrors the assembly lifecycle:

1. a proposal is written
2. WorldBuilder classifies it
3. BlockBuilder scopes it
4. InterfaceRegistry decides shared-contract promotion
5. Arbiter resolves contradictions
6. Archivist records accepted truth
7. PublicReadiness decides whether the result is publishable

This keeps the hub useful both for builders inside the repo and for people copying the repo as a starter system.
